# Прогнозирование кликабельности рекламы (CTR Prediction)

Проект по разработке предиктивной системы бинарной классификации для оценки вероятности клика по рекламному баннеру (Click-Through Rate) с использованием калиброванных линейных моделей.

## Архитектура и конфигурация решения
В рамках проекта исследованы и оптимизированы два базовых линейных классификатора:
* LogisticRegression — алгоритм с хорошей калибровкой из коробки.
* LinearSVC — алгоритм со сильным исходным искажением уверенности, исправленным с помощью честной изотонической регрессии на кросс-валидации (cv=5).

Для обработки высокоразмерных категориальных фич применено целевое кодирование (Target Encoding) с защитой от утечки данных.

## Метрики качества и калибровки
Обе финальные модели продемонстрировали высокую предсказательную способность и стабильность вероятностных выходов:

* Метрика качества (PR-AUC): Качество моделей кардинально выше базового уровня DummyClassifier. Метрика PR-AUC на тесте увеличилась практически в 1.86 раза — с 0.1721 (случайный классификатор) до 0.3189 для Логистической регрессии и 0.3208 для LinearSVC.
* Калибровка LinearSVC: Исходные метрики (Brier Score = 0.1638, ECE = 0.0179) после применения изотонической регрессии улучшились до ECE = 0.0066 (ошибка 0.66%), а Brier Score оптимизирован до 0.1316.
* Калибровка LogisticRegression: Дополнительная калибровка зафиксировала среднюю ошибку ECE на уровне 0.0060 (0.60%) при Brier Score = 0.1317.

## Ключевые драйверы кликабельности
Анализ весов моделей выявил ключевые факторы, влияющие на CTR:
* Наибольший положительный вклад вносят признаки контекста размещения, обработанные через Target Encoding: идентификатор категории C17 и идентификатор площадки site_id.
* Наибольший негативный вклад вносят категория C21_71 и баннерная позиция banner_pos_0, что указывает на снижение вероятности клика при данных условиях.

## Перспективы развития проекта

1. Генерация новых признаков (Feature Engineering):
* Продвинутые счетчики: Использование многомерных сглаженных счетчиков (парные комбинации, например, средний CTR для связки site_id + app_id).
* Поведенческие признаки: Добавление агрегатов по пользователям (количество кликов/показов данного пользователя за последние часы или дни) для выделения активных ботов или лояльной аудитории.
* Циклические временные признаки: Выделение часа суток и дня недели, преобразованных через синус и косинус (гармонические функции).

2. Переход к нелинейным моделям:
* Использование градиентного бустинга (CatBoost, LightGBM или XGBoost) позволит автоматически находить нелинейные комбинации признаков без ручной работы. Пакет CatBoost особенно эффективен для данных с обилием категориальных фич из коробки.

3. Работа со скрытыми взаимодействиями (Interaction Features):
* Применение кросс-признаков (например, комбинация device_model + banner_pos) вручную или через алгоритмы FM/FFM (Factorization Machines), созданные специально для разреженных рекламных датасетов.

4. Борьба со сдвигом данных (Data Drift):
* Реализация регулярного дообучения модели на скользящем окне свежих данных (например, обучение за последние 7 дней) и валидация строго по времени (Time-based split), а не случайным образом.
